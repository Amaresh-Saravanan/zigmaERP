# TDD Blueprint — MongoDB → MySQL Migration

**Companion to:** `PRD.md`, `Migration_Tracker.md`
**Principle:** tests are the migration's safety net. Write the contract test against the *current Mongo* backend first, watch it pass, then migrate the app and make it pass again against MySQL. If a test passes on Mongo and fails on MySQL, the migration broke something — that's the whole point.

---

## 1. Test Strategy

Layers, in order of value for a migration:

| Layer | What it protects | Tool | Target |
|-------|------------------|------|--------|
| **Golden API (contract)** | The frontend's contract — exact JSON in/out | `pytest` + DRF `APIClient` | 100% of live endpoints |
| **Model** | Field types, defaults, computed fields, constraints, soft delete | `pytest-django` | ≥80% |
| **Serializer** | Validation, nested writes, auto-code generation, ref resolution | `pytest` | ≥80% |
| **View / integration** | CRUD + permissions + envelope + pagination | `APIClient` | ≥70% |
| **Data migration** | Row counts, checksums, ref integrity post-ETL | standalone script + `pytest` | every collection |

**Stack:** `pytest` + `pytest-django` + `pytest-cov` + `factory_boy`. Drop `mongomock`. Use a real MySQL test DB (or SQLite for fast local unit runs — but run the **final gate on MySQL**, since `UniqueConstraint(condition=...)`, `db_table`, and collation differ).

**Golden-first workflow:**
```
1. Write golden test, run against Mongo backend → capture/assert current response.  [RED→GREEN on Mongo]
2. Migrate the app to ORM.
3. Run the SAME golden test against MySQL backend.                                   [must be GREEN]
4. Any diff = migration bug. Fix model/serializer, not the test.
```

## 2. Model Tests

Factories replace hand-built docs. One factory per model.

```python
# tests/factories.py
import factory
from factory.django import DjangoModelFactory
from inventory.models import Unit, Item

class UnitFactory(DjangoModelFactory):
    class Meta:
        model = Unit
    unit_name = factory.Sequence(lambda n: f'Unit-{n}')

class ItemFactory(DjangoModelFactory):
    class Meta:
        model = Item
    item_name = factory.Sequence(lambda n: f'Item-{n}')
    unit = factory.SubFactory(UnitFactory)
    # item_code auto-generated — don't set it here; assert it in the test
```

### What every model test must cover

```python
import pytest
from django.db import IntegrityError
pytestmark = pytest.mark.django_db

def test_unique_id_autogenerated():
    item = ItemFactory()
    assert item.unique_id                       # UUID present
    assert len(item.unique_id) == 36

def test_soft_delete_default_false():
    assert ItemFactory().is_deleted is False

def test_item_code_unique():
    ItemFactory(item_code='IT-001')
    with pytest.raises(IntegrityError):
        ItemFactory(item_code='IT-001')

def test_pit_volume_computed_on_save():
    pit = PitFactory(length=2, width=3, height=4)
    assert pit.volume == 24.0                   # save() computes L*W*H

def test_culling_compound_unique():
    CullingProcessFactory(entry_date=d, shift_type='1', cylinder_no='C1')
    with pytest.raises(IntegrityError):
        CullingProcessFactory(entry_date=d, shift_type='1', cylinder_no='C1')
```

**Must-test checklist per model:**
- [ ] `unique_id` auto-generated, unique
- [ ] `is_deleted` defaults `False`
- [ ] `created_at` / `updated_at` populate (`auto_now_add` / `auto_now`)
- [ ] Every `unique=True` field raises `IntegrityError` on collision
- [ ] Every compound unique constraint (culling `entry_date+shift_type+cylinder_no`, oven `entry_date+starting_time`) raises on collision
- [ ] Computed fields (`volume`, `net_weight`, `grand_total`, `fuel_consumption`, `running_hours`) compute in `save()`
- [ ] `on_delete=PROTECT` FKs block deletion of referenced rows
- [ ] `choices` reject invalid values
- [ ] M2M (`Tray` lists on `EggProcess`/`PitStatus`/`FrpTrayProcess`) attach/detach correctly
- [ ] `has_screen()` parses comma-separated `screens` (accounts)

## 3. Serializer Tests

The serializers carry the real business logic — auto-codes, ref resolution, computed fields, input sanitization. **Highest-priority tests.**

```python
def test_item_code_generation_sequential():
    UnitFactory(unique_id='u1')
    s1 = ItemSerializer(data={'item_name': 'A', 'unit': {'unique_id': 'u1'}})
    assert s1.is_valid(); i1 = s1.save()
    s2 = ItemSerializer(data={'item_name': 'B', 'unit': {'unique_id': 'u1'}})
    assert s2.is_valid(); i2 = s2.save()
    assert i1.item_code == 'IT-001' and i2.item_code == 'IT-002'

def test_item_code_ignores_non_IT_codes():
    # regression: seeded 'ITM-DEMO-1' must not reset counter to IT-001
    ItemFactory(item_code='ITM-DEMO-1')
    ItemFactory(item_code='IT-005')
    ...  # assert next generated == 'IT-006'

def test_item_unit_ref_resolves():
    UnitFactory(unique_id='u1')
    s = ItemSerializer(data={'item_name': 'A', 'unit': {'unique_id': 'u1'}})
    assert s.is_valid()

def test_item_unit_ref_unknown_rejected():
    s = ItemSerializer(data={'item_name': 'A', 'unit': {'unique_id': 'nope'}})
    assert not s.is_valid()
    assert 'unit' in s.errors

def test_supplier_label_sanitized_uppercase_alpha():   # 'ab1c!' -> 'ABC'
    s = SupplierSerializer(data={'supplier_name': 'X', 'label': 'ab1c!'})
    assert s.is_valid() and s.validated_data['label'] == 'ABC'

def test_dc_nested_items_and_grand_total():
    s = DCSerializer(data={'dc_number': 'DC-1', ..., 'items': [
        {'desc': 'A', 'qty': 2, 'rate': 100}, {'desc': 'B', 'qty': 1, 'rate': 50}]})
    assert s.is_valid(); dc = s.save()
    assert dc.items.count() == 2
    assert dc.grand_total == pytest.approx(250 * 1.18)   # incl 18% tax
```

**Serializer checklist:**
- [ ] Auto-code sequence + the non-`IT-` regression (already a documented bug fix in `_next_item_code`)
- [ ] Ref resolution: valid ref resolves, unknown/deleted ref rejected (`is_deleted=False` filter)
- [ ] Input sanitization: `label` (alpha upper), `contact_no` (digits), `gst_no` (upper), `email`
- [ ] Computed/read-only fields not writable from client (`volume`, `grand_total`, `item_code`, `unique_id`)
- [ ] Nested write: `DC`→`DCItem`, `EggProcess`→`addons`/`trays`
- [ ] Conditional-required: `CullingProcess.others_remarks` required when `work_done=='3'`; `PitStatus` fields per `org_status`
- [ ] Server-computed-if-omitted: `fuel_consumption`, `running_hours`, `net_weight`

## 4. View / Integration Tests

Test through `APIClient` — this is where the envelope, permissions, pagination, and soft delete all meet.

```python
@pytest.fixture

    r = APIClient().get('/api/items/'); assert r.status_code == 401

```

**View checklist (per viewset):**
- [ ] `list` paginated, `is_deleted=False` only, `?search=` filters, `?page_size=` capped at 100
- [ ] `create` 201 + envelope `msg='create'`; duplicate → 400 envelope
- [ ] `retrieve` unknown `unique_id` → 404 envelope
- [ ] `update` 200 + `msg='update'`; unknown → 404
- [ ] `destroy` soft-deletes, `msg='success_delete'`, row persists
- [ ] Permissions: each action requires its `required_screens` entry; missing → 403; no token → 401
- [ ] Lookup by `unique_id`, never by integer PK

## 5. Data Migration Tests

Run after the ETL script populates MySQL from a Mongo export.

```python
def test_row_counts_match(mongo_export, mysql_db):
    for coll, model in COLLECTION_MODEL_MAP.items():
        assert model.objects.count() == len(mongo_export[coll])

def test_all_refs_resolve():
    # no ForeignKey points at a missing row
    for item in Item.objects.all():
        assert item.unit_id is not None and Unit.objects.filter(pk=item.unit_id).exists()

def test_unique_ids_preserved(mongo_export):
    # unique_id is the external key — must survive ETL byte-for-byte
    mongo_ids = {d['unique_id'] for d in mongo_export['items']}
    assert set(Item.objects.values_list('unique_id', flat=True)) == mongo_ids

def test_embedded_docs_became_rows(mongo_export):
    for dc in mongo_export['dc']:
        obj = DC.objects.get(unique_id=dc['unique_id'])
        assert obj.items.count() == len(dc['items'])

def test_soft_deleted_preserved(mongo_export):
    # migrate deleted rows too — don't silently drop them
    assert Item.objects.filter(is_deleted=True).count() == \
           sum(1 for d in mongo_export['items'] if d['is_deleted'])
```

**Data-integrity checklist:**
- [ ] Row count per collection matches
- [ ] `unique_id` set identical (no regenerated UUIDs)
- [ ] Every FK resolves; zero orphans (except intentional `RejectImage.ticket_no` string ref)
- [ ] Embedded lists → correct child row counts
- [ ] Soft-deleted rows migrated, not dropped
- [ ] Checksum on a numeric column (e.g. `SUM(qty)`) matches per collection
- [ ] Dates land in correct tz (`USE_TZ=True`, `TIME_ZONE='UTC'`)

## 6. Coverage Targets & Config

```ini
# pytest.ini / setup.cfg
[tool:pytest]
DJANGO_SETTINGS_MODULE = config.settings
addopts = --cov=accounts --cov=inventory --cov=process --cov=reports --cov=core
          --cov-report=term-missing --cov-fail-under=80
```

| Area | Min | Rationale |
|------|-----|-----------|
| Models | 80% | Constraints + computed fields are correctness-critical |
| Serializers | 80% | Business logic lives here |
| Views | 70% | Envelope + permissions + pagination |
| Migration script | manual gate | Every checklist item green before cutover |
